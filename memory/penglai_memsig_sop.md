# 记忆签名与作废 SOP（蓬莱 · 让 markdown 记忆"会自我维护"）

蓬莱给 GA 的 L2 事实库加一层**轻量元数据**——让每条事实带来历，过期的标失效而非丢失。
**不上向量库、不加解析器、不维护任何计数器/运行态**（守 L0「禁易变运行态」）：签名是写在事实后面的
HTML 注释，GA 的 file_read 当普通文本照读，完全透明。

## 写入：每条长期事实带签名
在 L2（`global_mem.txt`）写/更新一条事实时，在该行末尾追加签名注释：

```
- 主力开发机走 Tailscale，节点名 mac-mini-dev  <!--@ time=2026-06-14 source=user imp=8 -->
```

- **time**：`YYYY-MM-DD`（今天）——这是**事实的时间戳，不是计数器**，新旧靠它比较。
- **source**：`user`（用户亲口说的）或 `AI`（你推断/总结的）。
- **imp**：重要度 1-10（缺省 5）。长期身份/关键偏好打高，琐碎打低。

> 不用全库序号/计数器：那是 L0 明禁的易变运行态，且给 markdown 强加 DB 语义与并发竞态。
> 排序与作废全靠 `time` + `source`，纯文本、无状态。

## 作废：新值覆盖旧值，但**不删**旧值（守 L0 神圣性）
当一条**新事实和已有事实矛盾**（同一实体、同一槽位——如"家庭住址""主力机节点名""常用邮箱"）：

1. **不要删旧条目**。给旧条目的签名追加 `superseded_by=<新条目的 time>`：
   ```
   - 旧地址：朝阳区 xx 路 12 号  <!--@ time=2026-03-01 source=user imp=6 superseded_by=2026-06-14 -->
   ```
2. 新条目正常带自己的签名（不带 superseded_by）。
3. **L1 指针/索引、问答都只认 current**（没有 superseded_by 的那条）；同槽位有多条 current 时取 `time` 最新。
4. 铁律：**`source=user` 的事实永远不被 `source=AI` 覆盖**——即便 AI 条目 time 更晚。用户亲口说的优先。

## 为什么这样做
- **治过期污染**：个人管家最致命的死穴是"记了旧偏好/旧地址还当真"。签名+作废让你能分辨哪条最新、哪条已失效。
- **可追溯、不丢信息**：旧值留档可回溯（"上个月还住朝阳"），符合 L0"记忆只能 patch、不能删改"的神圣性。
- **全程用 `file_patch` 最小修改**，改不动宁可不改。
