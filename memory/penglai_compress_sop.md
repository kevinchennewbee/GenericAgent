# 可追溯压缩 SOP（压缩/总结记忆必须留出处）

触发场景：整理/压缩/归并 L2、L3 记忆内容；把多条旧记录总结成一条；归档历史信息。
分工：L1 索引瘦身（删词、ROI 评估）→ 读 `memory_cleanup_sop.md`；本 SOP 管**压缩内容时怎么不丢真相**。

## 核心原则：压缩可以缩文字，不可以断来源
总结是有损的，错误会每轮复利。每条压缩后的结论必须能一步追溯回原文核对。

## 操作
1. **带出处指针**：压缩产物每条结论后缀来源，格式 `(源:文件#关键词)` 或
   `(源:L4_raw_sessions/<会话文件>)`。读者（未来的你）拿指针能直接 file_read 定位原文。
2. **先立新、验证、再动旧**：禁止"总结+删原文"一步完成。顺序：写出压缩版（带出处）→
   file_read 复核压缩版没有歪曲 → 才能精简原文。改不动宁愿不改。
3. **拿不准就保留**：某条信息无法确认是否还有用 → 原样保留或仅迁层（L2→L3），不删。
4. **L4 永不删**：L4_raw_sessions 是最后的真相来源，只读不动；压缩只发生在 L1-L3。

## 坑
- 把多条事实归并成一句"漂亮的概括"，丢了数字/路径/版本号 → 概括必须保留可执行细节。
- 出处写"之前的会话" → 等于没写，必须是能 file_read 的具体路径或可搜索的关键词。
- 压缩时顺手"纠正"了看着不对的旧记录 → 未经行动验证禁改，存疑就标注 `[待核]` 留原文。
